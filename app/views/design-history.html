Design history

This page shows the history of the design of the {{ serviceName }} service.

You can find links to the UR murals on the index of each version. The mural of round 4 for card sorting is linked on the main index page.

Version 4

  • Made changes to groupings and structure of triage following card sorting user research
  • Cut the 'Track the progress of a filing' option on first question page
  • Created new 'Technical support' option on first question page - previously included under 'Reporting an issue or making a complaint'
  • Split 'Filing for a company' into two groups: 'Company accounts and confirmation statement' and 'Updating company information'
  • Some content changes to first question page categories
  • Changes to groupings and structure following card sorting UR and v3 UR
  • Organised pages into folders and updated routing
  • User testing... round 5

Version 4 - round 2

  • Using V4 for a second round of testing but with unfamiliar/novice users and different scenarios
  • Minimal changes - all content, no major design or structural changes to triage process
  • Scenarios: reporting misuse of an address without permission

User testing round 4 - card sorting exercise

  • Card sorting exercise with 6 users to test the groupings and structure of the triage - no use of the prototype
  • To gain insight on how users would categorise CH topics

Version 3

  • Focused on creating and testing GOV Forms - created general enquiries form and Share Capital form
  • Link to share capital form preview
  • Link to general enquiries form preview
  • Note that Gov Forms are not versioned and we have since made updates, so the forms will/may have been updated since this round of user testing. You can see the forms in the state they were tested in the mural for this round.
  • Created Share Capital outcome pages and included reference numbers for phone number
  • User testing scenarios: share capital form has been rejected, something else - to test both forms. Not all users made it to the general enquiries form.

Version 2

  • Added sixth category to first question page - checking the progress of a filing or query
  • Outcome pages: added complain about a company
  • UR R2 scenarios: appeal a penalty, problem with signing in

Version 1 - October 2024

  • First version of triage style site, designed to be in keeping with SmartAnswers style after UX review work. SmartAnswers is a GDS publishing tool and would mean we could build the triage without significant dev resource. Alternatively, we would also be able to develop a service in this style if GDS couldn't or we decided we wanted to.
  • SmartAnswers consists of a start page, question pages, and outcome pages
  • First version had 5 categories (registering for a company; filing for a company; closing or restoring a company; penalties, fine or appeals; reporting an issue or making a complaint) on first question page and 'something else'
